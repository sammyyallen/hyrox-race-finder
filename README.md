# HYROX Race Finder

A Skyscanner-style search tool for HYROX events, with real ticket availability scraped every 15 minutes directly from HYROX's official vivenu ticketing platform.

## How it works

```
scraper.py  (runs every 15 min)
    │  fetches {seller}.hyrox.com/api/events/{event-slug}
    │  (same public endpoint the buy page uses in your browser)
    ▼
availability.json  (written to disk)
    │
    ▼
index.html  (reads the JSON on page load, overlays availability on the event cards)
```

## Files

| File | Purpose |
|------|---------|
| `scraper.py` | Python scraper — fetches ticket availability |
| `index.html` | Frontend — the search UI |
| `availability.json` | Generated by scraper — read by frontend |
| `.github/workflows/scrape.yml` | Free GitHub Actions deployment |

## Quick start (local / Raspberry Pi / VPS)

### 1. Install dependencies
```bash
pip install requests schedule
```

### 2. Run once to test
```bash
python scraper.py --once
```
Check `availability.json` was created and `scraper.log` for errors.

### 3. Run continuously (polls every 15 min)
```bash
python scraper.py
```

### 4. Serve the frontend
```bash
python -m http.server 8080
```
Open http://localhost:8080 in your browser.

## Free deployment via GitHub Actions

This runs the scraper every 15 minutes for free using GitHub Actions, stores `availability.json` in the repo, and serves the site via GitHub Pages.

### Steps

1. **Create a new GitHub repo** (public or private)

2. **Push all these files:**
   ```
   scraper.py
   index.html
   .github/workflows/scrape.yml
   ```

3. **Enable GitHub Pages:**
   - Settings → Pages → Source: Deploy from branch → `main` → `/` (root)
   - Your site will be live at `https://yourusername.github.io/your-repo-name`

4. **Enable Actions:**
   - Actions tab → enable workflows
   - The scraper runs automatically every 15 minutes

5. **`availability.json` will be committed** to the repo by the action after each run. The frontend reads it from the same directory.

### Cost: £0

GitHub Actions free tier gives 2,000 minutes/month. Each scraper run takes ~30 seconds × 4 runs/hour × 24h × 30 days = ~864 minutes/month. Well within the free limit.

## Updating event slugs

When HYROX adds new events or puts new events on sale, you need to update `EVENTS_CONFIG` in `scraper.py`:

1. Go to the HYROX event page on hyrox.com and click "Buy Tickets"
2. You'll land on `https://{seller}.hyrox.com/event/{event-slug}`
3. Copy the `seller` subdomain and `event-slug`
4. Add or update the entry in `EVENTS_CONFIG`

**Known seller subdomains:**
- `gb` → UK events
- `da` → Germany, Austria, Switzerland
- `france` → France
- `benelux` → Netherlands, Belgium
- `spain` → Spain
- `ireland` → Ireland
- `usa` → USA
- `canada` → Canada
- `australia` → Australia
- `worlds` → World Championships
- `baltics` → Latvia, Lithuania, Estonia
- `korea` → South Korea
- `japan` → Japan
- `hongkong` → Hong Kong

## Availability logic

The scraper maps vivenu's ticket categories to our divisions using pattern matching on the category name (e.g. "HYROX MEN | SATURDAY" → `mens-open`).

When a division runs across multiple wave days (e.g. Men runs Tuesday, Wednesday, and Thursday), we show the **best** status across all days — if any wave day still has tickets, we show "available" rather than showing "sold out" just because Tuesday is full.

Status meanings:
- 🟢 **Available** — tickets on sale, stock > 20
- 🟡 **Limited** — ≤20 tickets remaining
- 🔴 **Sold out** — no tickets left
- ⚪ **Unknown** — division offered but not yet on sale, or scraper hasn't fetched this event yet
